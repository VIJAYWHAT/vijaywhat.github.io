Let’s Talk JDBC Client

simple, fast, and productive.

Reality Check

Why not raw JDBC?

  • Most use JPA, Hibernate, or Spring Data
  • But Raw JDBC feels complex and repetitive.

Where’s the complexity?

JDBC flow

  • Write a query
  • Send to DB, get results

But we also manage:

  • Connection management
  • PreparedStatement handling
  • Result mapping
  • Exception handling
  • Transaction management

What Makes Spring JPA or Spring JDBC Better?

Why Spring JPA?

  • No SQL needed
  • No connection handling
  • But... performance tuning is complex

Why Spring JDBC Client?

  • You only write SQL
  • Spring manages everything else
  • Great control over performance

What if we build our own JDBC client?

2 Strong Reasons

  • You might not use Spring in all projects
  • You want a lightweight, custom abstraction
